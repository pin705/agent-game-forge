# Recipe — Parallax background layers

Implements N-layer scrolling backgrounds with per-layer parallax factor,
opacity, and horizontal repeat. Standard for side-scroll action games to
create depth without tilemap layers.

## When to use

- Side-scroll action / platformer / runner where camera scrolls horizontally
- Want depth feel beyond a single background image
- Generated maps split into 3-5 layers (sky / far / mid / near / foreground)
- Each layer has its own parallax speed (0.05 for distant sky, ~1.0 for foreground)

## When NOT to use

- **Single-screen battlefield** (TD / arena / boss room) — use
  `background: { image: ... }` single-bg path, not layers
- **Top-down camera** — parallax doesn't make sense top-down; use a single
  `background.image`
- **Tilemap-based scrolling** (Pokemon, Stardew) — use TileMap instead

## Files this affects

- `src/parallax.js` — main module (~20 LOC)
- `src/render.js` — calls `drawParallax(ctx, level)` before platforms
- `data/<level>.json` — per-level `layers[]` array
- `assets/maps/<level_id>/sky.png / far_bg.png / mid_bg.png / near_bg.png` —
  layer images (generated by `generate2dmap` with `map_mode: side_scroll_mode`)

## Pattern

### 1. Level JSON shape

```json
"layers": [
  { "id": "sky",      "image": "assets/maps/border_road/sky.png",      "parallax": 0.03, "opacity": 1.0,  "zIndex": 0, "repeatX": true },
  { "id": "far_bg",   "image": "assets/maps/border_road/far_bg.png",   "parallax": 0.18, "opacity": 0.72, "zIndex": 1, "repeatX": true },
  { "id": "mid_bg",   "image": "assets/maps/border_road/mid_bg.png",   "parallax": 0.55, "opacity": 0.8,  "zIndex": 2, "repeatX": true },
  { "id": "near_bg",  "image": "assets/maps/border_road/near_bg.png",  "parallax": 0.98, "opacity": 0.84, "zIndex": 3, "repeatX": true }
]
```

Each layer:
- `id` — semantic name (required for editor + ref)
- `image` — relative path
- `parallax` — scroll factor (0 = static sky, 1 = scrolls 1:1 with camera)
- `opacity` — 0..1 alpha (use < 1 for atmospheric haze on far layers)
- `zIndex` — render order, ascending (sky farthest back)
- `repeatX` — tile horizontally? true for short layer image tiling across
  a long level; false for layer that spans the whole mapSize

### 2. Renderer

`drawParallax(ctx, level)` in `parallax.js`:
- Sort layers by zIndex ascending (lowest first = farthest back)
- For each layer:
  - `const offset = -((state.camera.x * scroll) % img.width)` — wrap calc
  - Tile horizontally from `offset - img.width` to `VIEW.w + img.width`
  - `ctx.globalAlpha = layer.opacity ?? 1`

### 3. Camera

`camera.js` `updateCamera(dt)`:
- Follows player x with lookahead (player facing × lookahead value)
- Clamps to `[0, mapSize.width - VIEW.w]`
- Vertical follow optional (most platformers keep y locked)

## Adaptation knobs

| Knob | Where | Effect |
|---|---|---|
| `parallax` per layer | level JSON | 0 = no scroll (sky), 1 = scrolls with player |
| `opacity` per layer | level JSON | Atmospheric depth — distant haze, fog wash |
| Number of layers | level JSON | 3 = minimum (sky + bg + foreground); 5 = max sensible |
| `zIndex` order | level JSON | Back to front |
| `repeatX` | level JSON | True for tileable strip; false for full-mapSize wide image |

## Asset pipeline

Layers come from `generate2dmap` with `map_mode: side_scroll_mode`:
- Single skill call produces 4 layer PNGs (sky/far/mid/near)
- Each layer image is the level's full width (e.g. 2560px) or a tileable
  strip (e.g. 1280px wide with seamless edge)
- Reference: convention `generate2dmap` skill, `references/scene_mode.md`
  and `references/side_scroll_mode.md`

## Common mistakes

1. **All layers `parallax: 1`** — no depth feel; might as well be one image.
   Use 0.05–0.3 for sky/far, 0.4–0.6 for mid, 0.8–1.0 for near.

2. **Layer image too narrow + `repeatX: false`** — gaps appear at edges
   when camera scrolls. Either set repeatX or generate full mapSize-wide.

3. **`opacity: 1.0` on every layer** — flat look. Use 0.6–0.85 on distant
   layers to suggest atmospheric haze. (Sky usually stays at 1.0.)

4. **Wrong zIndex** — far layer in front of player. Always sort sky=0,
   foreground = highest, player draws AFTER all parallax.

5. **`parallax > 1`** — layer scrolls faster than player. Reserved for
   special FX (foreground vines whipping by); confusing on normal bg.
